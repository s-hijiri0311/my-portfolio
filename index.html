Line Bot

平山シャトルとは

私は大学生入学と同時にバドミントンサークルSmash!!へ加入し、2017年8月から1年間100人以上のメンバーの代表を務めました。その後、2018年10月にバドミントンサークル平山シャトルを10人のメンバーと共に発足し現在20人程のメンバーで活動を続けております。

Line Bot 開発の背景

平山シャトルでは少数での活動が多いため、活動日の連絡をまめに行い、参加者を確保する必要があります。また、どの活動日にどのメンバーが参加するかを確認する手段が必要です。 このような連絡等を自動化するために平山シャトル専用のLine Bot"ヒラヤマン"を開発しました。ヒラヤマンの機能は大きく分けて3つあり、これらのシステムはGoogle Apps Script(GAS)で開発しました。GASを用いることでメンバーが持っているgoogleアカウントをそのまま利用することが可能であり、ドキュメントやスプレッドシートとの連携が簡単であるからです。サーバーを必要とせずにWEBアプリを開発できるのも利点であるためGASを採用しました。開発期間は実用するまでに1ヶ月ほどでしたが、現在も改良を続けております。



日程調整機能

googleアカウントのスプレッドシートとカレンダーをGASによって結びつけを行うことでカレンダーに登録した活動場所の情報と日時をcalendar.getEventsメソッドを用いることで実現しました。メンバーのリストは1行目に予め入力されておりますが、２行目以降の活動日と活動場所等の情報は毎回の活動日が終わる時間にトリガーを設定し自動で更新される仕組みになっています。過去の情報は不要になるため、毎回の活動が終わるたびの更新によって常に一番上が最近の活動予定となるように実装しました。スプレッドシート内にも関数が入力される仕組みになっているため作成された予定表を見ることで参加者の人数等も把握できる仕組みになっています。

予算管理機能

上記のように自動作成される予定表を活用することで予算の管理も自動で実現することが可能となりました。具体的には、本サークルでは毎月の参加回数に応じて月毎の支払い金額が変わるシステムになっているので参加予定表に入力させている記号の数をカウントすることで月毎の金額を自動計算し、各個人の金額を表示するシステムを導入しました。これにより多くのメンバーの参加回数や集金の確認をする必要がなくなり、集金を行なったら○を入力するだけでサークル全体の予算も計算されるようにスプレッドシート に関数を記述しています。

自動連絡機能

最後に上記の日程調整等のWEBアプリを利用したLine Botです。サーバーレスであるGASを用いて自動送信コードをjson形式で記述しLine DeveloperのWeb hook URLに追加することで実現しました。日程調整のスプレッドシートから次回の活動場所と予定日、参加予定記号がついている参加者の名前を取得しサークルのライングループへ活動日の前日や月の初め等に自動送信します。次回予定や来月の予定が知りたい場合は"次回"や"来月の予定"など知りたい情報をBotに送信することで返信が自動で返ってきます。他にも役に立たない機能等が実装されていて、"ヘルプ"と送ることで何ができるのか確認できるようにしてあります。

Android App

Math Logic Button

Google Play StoreにリリースしたMath Logic Buttonについてです。 こちらはAndroid Studioとkotlinを用いて2週間ほどで友人とのハッカソン形式で個人開発しました。 これはゲームアプリなのですが、3色のボタンが用意させていてそれぞれに簡単な数学的計算能力が存在しています。プレイヤーはその３つのボタンの持つ機能を見つけ出し目標となる数字へ最短の手で到達することを目指します。本ゲームアプリでは画面遷移のためのNavigationやゲームのプレイデータを保存するpreferenceなどの技術が採用されております。上の画像は実際のアプリ画面になっていて左がオープニング画面（オープニングのデザインも少し意識しました）、GAMESTARTを押した後のステージセレクト画面が真ん中、HELPを押した場合のチュートリアルが右の画面になります。実際にリリースすることを目標にしていたのでルール説明を充実させることを特に意識しました。

ゲーム画面

ゲーム画面とヒントについてです。 実際のプレイでは、中央上の赤い数字が到達するための数字でそのしたの灰色の数字が現在の数字です。画面左上の赤い数字は到達するまでの最短の手数になっていてその下の青い数字は現在の手数になっています。プレイヤーは３つのボタンを押して試行錯誤することでそれぞれのボタンの機能を推測していくことを想定していますが実際にリリースされているゲームを真似して作りたかったためヒントボタンとGoogle AdSenseを採用しました。実際にリリースされているアプリでは広告を視聴することで右のような画面に遷移し、それぞれのボタンの機能を知ることが可能です。

クリア画面

flagmentを利用することでバックボタンを用いることで一手戻ることが可能ですが手数はリセットされない仕様にしてあるため、最短で到達できない可能性があります。最短で到達できた場合は次のステージが解放され最短手以外で到達した場合にはTry Againとなります。

ゲームとしてのこだわり

ゲームアプリの開発により特にこだわった点が裏ボタンとデザイン性です。ステージ６をクリアした場合にはステージ７は解放されずオープニング画面に変化が起きます。また、オープニング画面の三色のバーがビューからボタンへと変化しているためこれらのボタンを順に押すことでfinal stageが解放されるようにしました。ステージは7つにしたのでステージボタンを7色にしたのもこだわりの一つです。 数学的なアルゴリズムの問題なので最短手を計算するのに少し苦労しましたが、ゲームアプリの開発は自分がやっても楽しめるようなゲームを目指し、実際に遊ぶこともできたので楽しかったです。

Raspbery Pi

今年の正月に入る頃にスマートリモコンに興味を持ちました。理由としては寒い冬や暑い夏に家に帰る前にエアコンを操作できたら便利だと感じたことや、家で生活する上で布団の位置から照明のリモコンが届かないことから開発を考えました。スマートリモコンは外からでも操作することができる方が良いと考えていたためwifiに接続可能なraspbery pi によって開発しました。授業でラズパイのGPIOについては学んだことがあったため回路にはそこまで苦労せずに抵抗と受信モジュール、送信モジュールを用いることで実現することができました。ラズパイの中ではpythonで赤外線の送受信を操作するpigpioライブラリを用いました。外からでも操作を行えるようにするためにslack botであるhubotを用いて特定のメッセージに対して操作を実行するように実装しました。最後にslackへのメッセージを自動化しなければ面倒なのでhttp通信を行なってslackのapiを叩くslackへの自動送信アプリを開発し、スマートリモコンを完成させました。赤外線の受信モジュールを利用することで赤外線で操作できる家電はなんでも操作可能となっています。簡単ではありますがいかにslack自動送信アプリの画面を載せます。こちらのアプリではhttp接続を行うFuelとエアコンの設定温度を記憶するためのpreferenceを用いています。

Research

大学での研究は深層学習を用いた物体検出器を利用した作業認識AIの開発を企業との共同研究として行なっております。学部時代の卒業論文では物体検出器YOLOv3を用いて固定カメラから撮影した特徴のある作業のアノテーションを行い学習させることで姿勢に特徴のある作業の検出と分類を行うことができました。このシステムの開発にはpythonを用いており機械学習によく用いられるライブラリNumpyやmatplotlibなどの他に深層学習に用いられるライブラリであるtensorflowやkerasを利用しています。今年6月には人工知能全国大会(JSAI)の国際セッションでHuman Action Classification using Object Detection and Voice Recognition in Industrial Environmentsの題で発表をしました。JSAIで発表した提案手法はYOLOv3での物体検出から検出された作業は姿勢のみしか考慮していないため、特徴的な作業音を発生させる作業には以下の画像のように音声データにフーリエ変換を用いることで作業中の時間を計算することができ、精度を向上させることができるという手法になります。
